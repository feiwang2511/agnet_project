# 30/60/90 天行动计划

## 背景

错题本 AgentOS 已完成初始搭建：Knowledge 系统、三层治理、Delivery Engine、Hooks、Eval 和 Loop 配置均已就位。以"拍照识别"功能完成了一次完整 SDLC 实弹。

本计划将课堂产物带入真实工作，逐步验证和放权。

## 30 天：验证 + 积累

### 目标

在一个真实项目中跑 AgentOS，积累足够 corrections 进行首次蒸馏。

### 行动项

- [ ] 将 AgentOS 模板接入一个真实项目仓库
- [ ] 用 feature profile 完成 2-3 个功能任务
- [ ] 每次 session 结束确认 corrections.log 被正确捕获
- [ ] 积累 10+ 条 corrections
- [ ] 执行首次蒸馏：上提/下沉/毕业
- [ ] 跑 eval，记录 baseline 分数
- [ ] 验证 Health Check 能发现文档过期

### 成功标准

- Eval 分数 >= 70%
- corrections 覆盖 3 种以上类别
- 至少完成一次蒸馏，governance 行数不增长

## 60 天：渐进放权 + Loop

### 目标

开始让 Agent 在受控条件下自动循环处理简单任务。

### 行动项

- [ ] 开放 bugfix profile 的自动 loop
- [ ] 配置熔断条件并验证触发
- [ ] 观察 Gate 通过率和升级频率
- [ ] 完成第二次蒸馏
- [ ] 尝试 refactor profile 的有条件自动化
- [ ] Eval 分数稳定 >= 80%
- [ ] 检查 Loop 成本治理是否有效

### 成功标准

- bugfix 任务可全自动完成（G1-G4 无人工介入）
- 熔断条件至少被验证触发过一次
- Eval 分数 >= 80%
- 人工升级频率相比 30 天下降

## 90 天：跨项目复用 + 成熟

### 目标

AgentOS 模板可迁移到第二个项目，跨项目 principles 开始沉淀。

### 行动项

- [ ] 接入第二个项目，复用 governance 和 engine 框架
- [ ] 识别跨项目通用 principles vs 项目特定 rules
- [ ] 分离通用模板和项目特定配置
- [ ] feature profile 的 PLAN 阶段尝试 L2 自查
- [ ] 建立跨项目 eval 对比机制
- [ ] 输出 AgentOS 使用手册（面向团队其他成员）

### 成功标准

- 第二个项目 eval 分数首次 >= 70%
- 跨项目 principles 3-5 条，稳定不变
- 团队至少 1 人能独立使用 AgentOS
- 月均蒸馏 1 次，governance 总行数持续可控

## 风险

| 风险 | 缓解 |
| --- | --- |
| corrections 积累不足 | 前 30 天有意观察 Agent 偏差，不急于纠正 |
| governance 膨胀 | 每次蒸馏强制检查行数，退休低效规则 |
| Loop 失控 | 初期只开放 bugfix，熔断条件保守 |
| 跨项目适配困难 | 先分离通用/特定，再逐步泛化 |
| 团队不接受 | 用 eval 分数证明效果，不强推流程 |
