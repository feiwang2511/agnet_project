# 错题本 - 改进方向

## 优先级

1. 识别结果可靠性：先保证 schema 完整、置信度可判定、低置信度不污染正式数据。
2. 首次使用体验：拍照到识别结果展示小于 5 秒。
3. 知识点归类质量：让复习计划能围绕知识点稳定生成。
4. 可验证交付：每个功能完成时必须有测试和主动破坏记录。

## 已知 Tech Debt

- 识别准确率 baseline 尚未建立。
- Bedrock 返回解析和业务 schema 校验需要隔离。
- 错误处理格式尚未统一。
- 前端确认流程尚未设计。
- 测试覆盖率未知。

## 当前改进假设

- 如果低置信度结果都进入待确认，复习计划的数据污染会明显降低。
- 如果识别结果 schema-first，前后端联调失败会减少。
- 如果每次任务结束都记录 corrections，AgentOS 的治理规则会逐步变少但更有效。

## 禁止事项

- 不要绕过 schema 校验直接返回 AI 原始输出。
- 不要把 Bedrock SDK 细节泄漏到业务层。
- 不要在代码、日志或文档中写入真实 API key、访问凭证或用户隐私图片内容。
- 不要新增 ORM；当前数据访问应遵循 DynamoDB 访问模式。
- 不要在没有测试和验证记录时声称功能完成。

## 需要观察的信号

- 低置信度结果是否被正确标记为待确认。
- `question_text`、`confidence`、`knowledge_points` 是否总是存在。
- Agent 是否跳过 EVALUATE/PLAN 直接进入 BUILD。
- 同类 correction 是否重复出现 3 次以上。
